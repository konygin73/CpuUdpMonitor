# CpuUdpMonitor

Простая система для удаленного мониторинга загрузки процессора в реальном времени. Состоит из легковесного агента на Си и графического сервера на Qt.

## Структура проекта

*   **CpuUdpClient**: Консольный агент для Linux (C11). Считывает данные из `/proc/stat` и отправляет их по протоколу UDP. Работает в многопоточном режиме.
*   **CpuUdpServer**: Графическое приложение (Qt 5/6 + QCustomPlot 2). Принимает UDP-пакеты и строит графики загрузки для каждого ядра процессора отдельно.

## Сборка и запуск

### 1. Клиент (Агент)
Требуется ОС Linux и компилятор GCC/Clang.

```bash
cd CpuUdpClient
mkdir build && cd build
cmake ..
make
# Запуск: ./cpu_monitor <IP_СЕРВЕРА> <ПОРТ>
./cpu_monitor 127.0.0.1 9999
```

### 2. Сервер (Визуализатор)
Требуется установленный Qt (5.12+) и библиотека QCustomPlot (уже включена в проект).

1. Откройте `CpuUdpServer/CpuUdpServer.pro` в Qt Creator.
2. Соберите и запустите проект.
3. По умолчанию сервер слушает порт `9999`. Если нужно изменить порт, передайте его аргументом командной строки:
   `./CpuUdpServer 8888`

## Особенности реализации
*   **Низкая нагрузка**: Клиент использует системный вызов `pread` и быстрый парсинг `/proc/stat` без тяжелых функций.
*   **Многопоточность**: Сбор данных и сетевой обмен в клиенте разнесены по разным потокам (pthread).
*   **Масштабируемость**: Сервер автоматически определяет количество ядер в пришедшем пакете и динамически создает нужное количество графиков.
*   **Точность**: Расчет ведется по дельтам накопленных значений тиков процессора между итерациями.
